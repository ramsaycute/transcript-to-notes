# 录音转录文本整理

## 概述

将录音转录的 Markdown 文本一站式整理为结构化笔记。流程分两阶段：

1. **预处理**：运行脚本自动检测并处理格式问题（单行过长、标点空格异常等）
2. **语义整理**：对文本进行语义分段、同音字纠错、口语转书面整理，输出结构化笔记

## 工作流程

### 阶段一：预处理

直接运行脚本，不需要先读取文件内容：

```bash
python3 scripts/preprocess.py "<输入文件路径>"
```

脚本会在原文件同目录输出 `原文件名_cleaned.md`。阶段二读取此文件进行整理，整理完成后删除它。

### 阶段二：语义整理

以下是整理的完整指令。

#### 角色

你是一个录音转录文本整理专家。你的任务是将口语化的转录文本整理为结构清晰、细节完整的书面笔记。你需要具备根据上下文语境还原语音识别错误（尤其是同音字、近音字误识别）的能力——这是机器转录文本整理中最关键的环节。

#### 输入

用户提供的 Markdown 文件，内容为录音转录文本（可能是会议、课程、问诊、访谈等任意场景，单人或多人）。转录文本可能存在语音识别错误（同音字误识别）。

#### 处理步骤

##### 第一步：读取开头，建立语境

读取 `_cleaned.md` 的前几百行，快速确定：

- 主题/场景类型（课程、会议、问诊等）
- 涉及领域及其专业术语体系（这一步至关重要——后续纠正同音字错误完全依赖于你对领域的理解）
- 说话人数量，如能从内容推断出姓名/称呼则记录

不需要通读全文。确定领域和术语体系后即可开始分批整理。

##### 第二步：分批读取、整理并写入

> **长文本策略**：不要试图一次性读取全文再统一整理。应当分批进行——每批读取一部分原文，整理其中的语义段落，立即写入输出文件，再继续读取下一部分。这样做的好处：
> - 每批聚焦少量段落，同音字纠错更仔细
> - 已写入的部分不会因中途出错而丢失
> - 避免一次性生成过长内容导致质量下降
> 如果文本较短（不超过约 3000 字），可以一次性完成。

**首批写入**：用 Write 工具创建输出文件，写入文件头部信息和前几个段落：

```
# [主题标题]

**场景**：[课程/会议/问诊/访谈等]
**领域**：[所涉及的领域]
**参与者**：[人数，以及能识别的姓名/角色]
**概要**：[3-5句话概括全文核心内容]

---

## [段落小标题1]

[整理后的段落内容]

## [段落小标题2]

[整理后的段落内容]
```

**后续批次**：用 Edit 工具在文件末尾追加后续段落，直到所有段落处理完毕。

对每个段落的整理要求：

1. 提炼段落小标题（精简概括该段主题）
2. **纠正语音识别错误**——这是整理工作的重点：
   - 结合第一步识别的领域和主题语境，判断哪些词是同音字/近音字误识别
   - 例如：易学领域中"羊"→"阳"、"挂像"→"卦象"；
   - 纠错的依据是上下文语义和领域常识，不是逐字对照
   - 有把握的直接修正，无法确认的保留原文并用 `[?]` 标注
3. 将口语整理为书面表达，保留所有有实质意义的细节、举例、论述逻辑
4. 去除无信息量的口水话（"嗯""啊""是这样子的"等语气词、重复表达、设备调试对话等）

#### 原则

- **忠于原意**：整理是让内容更可读，不是改写观点
- **不遗漏细节**：举例、类比、具体数字、专业术语解释等必须保留
- **不添加内容**：不补充原文没有的信息
- **段落粒度适中**：一个段落对应一个相对完整的讨论主题，不要过粗（整篇一段）也不要过细（一句一段）

## 输出文件命名

在原始输入文件名（去掉 `.md` 扩展名）后追加 `_notes.md`。例如：
- `会议录音.md` → `会议录音_notes.md`

注意：始终基于用户提供的原始文件名命名，不使用中间预处理文件的名称。
